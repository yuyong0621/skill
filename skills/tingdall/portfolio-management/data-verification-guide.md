# 数据验证操作指南

## 大盘指数数据验证流程

### Step 1: 明确报告日期

在搜索前，先确认报告日期：
```
报告日期：2026年3月6日（星期五）
数据需求：2026年3月6日收盘数据
```

### Step 2: 多源搜索验证

**必须执行至少2次独立搜索：**

```bash
# 搜索1：新浪财经
搜索词："3月6日 上证指数 收盘 涨跌幅 新浪财经"

# 搜索2：东方财富
搜索词："上证指数 今日收盘 3月6日 东方财富"

# 搜索3：证券时报（备用验证）
搜索词："沪指 深成指 3月6日 收盘 证券时报"
```

### Step 3: 数据一致性检查

| 检查项 | 通过标准 | 处理方式 |
|--------|---------|---------|
| 日期一致性 | 所有数据标注日期与报告日期一致 | 不一致则排除 |
| 时间一致性 | 收盘数据优先于盘中数据 | 盘中数据标注"午间" |
| 数值一致性 | 两个来源涨跌幅差异<0.5% | 差异大则搜索第三个来源 |
| 逻辑一致性 | 涨跌幅与点位变化方向一致 | 不一致则重新核实 |

### Step 4: 数据输出格式

**正确格式：**
```markdown
### 今日市场回顾

**数据来源：** 新浪财经、东方财富（2026-03-06 15:00收盘）

| 指数 | 收盘点位 | 涨跌幅 | 成交额 | 备注 |
|------|---------|--------|--------|------|
| 上证指数 | 4,118.68 | +0.25%（午间）/ +0.38%（收盘） | 2.39万亿 | 低开高走 |
| 深证成指 | 14,150.45 | +0.80%（午间）/ +0.59%（收盘） | - | 冲高回落 |
| 创业板指 | 2,XXX.XX | +0.85%（午间）/ +0.38%（收盘） | - | 震荡整理 |
```

**错误格式（避免）：**
```markdown
❌ 上证指数：~4,082点（-0.98%）  ← 无数据来源，无时间戳
❌ 深证成指：~13,918点（-0.75%） ← 可能混用历史数据
```

## 常见数据错误案例

### 案例1：日期混淆

**错误场景：**
- 搜索"上证指数 今日收盘"返回了3月3日的历史文章
- 文章标题："3月3日收盘，上证指数下跌1.43%"
- 误以为是3月6日数据

**正确做法：**
- 检查文章发布时间戳
- 确认标注"2026-03-06"的数据
- 排除3日以上的历史数据

### 案例2：盘中/收盘混淆

**错误场景：**
- 看到"午间收盘，沪指跌1.43%"
- 未注意到"收盘涨0.38%"的更新
- 报告写"午后有所回升"（实际是收盘上涨）

**正确做法：**
- 明确标注"午间"或"收盘"
- 优先使用15:00收盘数据
- 若使用盘中数据，必须说明"截至午间"

### 案例3：来源单一

**错误场景：**
- 只搜索了一个来源
- 该来源数据有误或过时
- 无交叉验证导致错误

**正确做法：**
- 至少2个独立来源验证
- 差异大时引入第三个来源
- 优先选择官方交易所数据

## 数据时效性检查清单

### 搜索前检查

- [ ] 确认报告日期（年-月-日）
- [ ] 确认数据类型（收盘/盘中/开盘）
- [ ] 准备2-3个搜索关键词组合

### 搜索中检查

- [ ] 检查每条结果的发布时间
- [ ] 排除3日以上的历史数据
- [ ] 区分"预测"与"实际"数据
- [ ] 识别"复盘"与"实时"数据

### 搜索后检查

- [ ] 对比2个来源的数据一致性
- [ ] 检查涨跌幅与点位变化是否匹配
- [ ] 确认成交额数据合理性
- [ ] 标注所有数据的来源和时间

### 输出前检查

- [ ] 表格中每个数据都有来源标注
- [ ] 区分了盘中数据和收盘数据
- [ ] 没有混用不同日期的数据
- [ ] 涨跌方向与数值一致

## 数据错误应急处理

### 发现数据不一致时

1. **停止写入报告**
2. **搜索第三个来源验证**
3. **优先选择：**
   - 官方交易所数据 > 财经媒体数据
   - 收盘数据 > 盘中数据
   - 多源一致的数据 > 单源数据
4. **在报告中注明：** "数据存在争议，以XX来源为准"

### 无法找到当日数据时

1. 使用最近一个交易日的数据
2. 明确标注："截至最新可获得数据（2026-XX-XX）"
3. 说明原因："当日收盘数据尚未公布"
